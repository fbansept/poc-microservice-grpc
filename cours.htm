01Qu'est-ce que gRPC ?

    
gRPC est un framework RPC haute performance
 de Google, basé sur HTTP/2 et Protocol Buffers. Comparé à REST+JSON, il
 est typiquement 5 à 10× plus rapide en sérialisation.


    

      
CritèreREST + JSONgRPC + Protobuf

      
        
Protocole transportHTTP/1.1HTTP/2

        
Format sérialisationJSON (texte)Protobuf (binaire)

        
Contrat d'APIOpenAPI (optionnel).proto (obligatoire)

        
Génération de codeManuel ou SwaggerAutomatique (plugin Maven)

        
StreamingSSE / WebSocket séparéNatif (4 modes)

        
PerformanceRéférence~10× plus rapide

      
    


    

      💡
      
Quand utiliser gRPC ? Pour les communications internes
 entre microservices (haute fréquence, faible latence). REST reste 
préférable pour les API publiques consommées par des navigateurs ou des 
tiers.

    


    
Les 4 modes de communication gRPC

    

      
Unary
Requête / Réponse
Un appel → une réponse. Le mode le plus courant dans les microservices. C'est ce que nous utilisons ici.

      
Server Streaming
Réponse multiple
Un appel → flux de réponses. Utile pour les exports de données en temps réel.

      
Client Streaming
Envoi multiple
Flux de requêtes → une réponse. Utile pour les uploads ou l'agrégation de données.

      
Bidirectionnel
Full Duplex
Flux dans les deux sens simultanément. Pour les systèmes de chat, jeux en ligne, etc.

    

  


  
  

    
02Fichier Protobuf — le contrat

    
Le fichier .proto est la pièce centrale. Il définit le contrat entre les deux microservices : services, méthodes, et messages. Il est partagé entre stock et vente.


    

      📁
      
Emplacement dans ce projet : Le fichier stock.proto se trouve dans common/src/main/proto/ — le module Maven partagé (voir section 03).

    


    

      

        PROTO
        src/main/proto/stock.proto
        
      

      
// Syntaxe proto3 — obligatoire
syntax = "proto3";

package stock;

// Options Java de génération
option java_package         = "com.example.grpc.stock";
option java_multiple_files  = true;
option java_outer_classname = "StockProto";

// ─────────────────────────────────────────
// Définition du service gRPC
// ─────────────────────────────────────────
service StockService {
    // Vérifier le stock disponible pour un produit
    rpc VerifierStock    (StockRequest)      returns (StockResponse);

    // Décrémenter le stock lors d'une vente
    rpc DecrementerStock (DecrementerRequest) returns (DecrementerResponse);
}

// ─────────────────────────────────────────
// Messages de requête
// ─────────────────────────────────────────
message StockRequest {
    string produit_id = 1;   // ⚠ numéros immuables en production
}

message DecrementerRequest {
    string produit_id = 1;
    int32  quantite   = 2;
}

// ─────────────────────────────────────────
// Messages de réponse
// ─────────────────────────────────────────
message StockResponse {
    string produit_id          = 1;
    int32  quantite_disponible  = 2;
    bool   disponible           = 3;
}

message DecrementerResponse {
    bool   succes           = 1;
    int32  quantite_restante = 2;
    string message          = 3;
}

    


    

      ⚠️
      
Numéros de champs immuables :
 Les numéros (= 1, = 2…) identifient les champs dans le flux binaire 
Protobuf. Ne jamais les modifier ni les réutiliser une fois le service 
en production, sous peine d'incompatibilités silencieuses entre 
versions.

Pourquoi mvn install et non mvn compile ?

    
C'est la question clé avec un projet multi-modules. compile ne suffit pas dès qu'un module dépend d'un autre.


    

      
CommandeCe qu'elle faitRésultat pour common

      
        

          mvn compile
          Compile les sources → .class dans target/classes/
          Pas de JAR — stock et vente ne peuvent pas le résoudre
        

        

          mvn package
          Compile + crée le JAR dans target/
          JAR créé mais non publié — Maven ne le trouve pas en dépendance
        

        

          mvn install
          Compile + package + copie le JAR dans ~/.m2/repository/
          JAR disponible pour tous les modules qui en dépendent
        

      
    


    
Quand Maven résout la dépendance edu.ban7:common:0.0.1-SNAPSHOT dans stock/pom.xml, il cherche le fichier ici :

    

      

        CHEMIN
        Dépôt local Maven
      

      
~/.m2/repository/edu/ban7/common/0.0.1-SNAPSHOT/common-0.0.1-SNAPSHOT.jar

    


    
Ce fichier n'existe que si mvn install a été lancé. Avec compile seul, Maven échoue avec :

    

      

        ERREUR
        Console
      

      
Could not resolve dependencies for project edu.ban7:stock:jar:0.0.1-SNAPSHOT:
  Could not find artifact edu.ban7:common:jar:0.0.1-SNAPSHOT

    


    

      ✅
      

        Règle pratique : Depuis la racine du projet, toujours utiliser mvn install. Maven respecte l'ordre des <modules> : il compile et installe common en premier, puis stock et vente trouvent le JAR dans ~/.m2 au moment de leur propre compilation.


        En IntelliJ : panneau Maven → sélectionner le module racine → Lifecycle → install.

Implémentation du service gRPC

    

      

        JAVA
        stock/src/main/java/edu/ban7/stock/grpc/StockService.java
        
      

      
package edu.ban7.stock.grpc;

import com.example.grpc.stock.*;
import io.grpc.Status;
import io.grpc.stub.StreamObserver;
import net.devh.boot.grpc.server.service.GrpcService;

import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;

@GrpcService
public class StockService extends StockServiceGrpc.StockServiceImplBase {

    // Simulation BDD — thread-safe pour les appels gRPC concurrents
    private final Map<String, Integer> stocks = new ConcurrentHashMap<>(Map.of(
            "P001", 100,
            "P002", 50,
            "P003", 0   // rupture de stock
    ));

    @Override
    public void verifierStock(StockRequest request,
                               StreamObserver<StockResponse> responseObserver) {
        String produitId = request.getProduitId();
        Integer quantite = stocks.get(produitId);

        if (quantite == null) {
            responseObserver.onError(Status.NOT_FOUND
                    .withDescription("Produit introuvable : " + produitId)
                    .asRuntimeException());
            return;
        }

        responseObserver.onNext(StockResponse.newBuilder()
                .setProduitId(produitId)
                .setQuantiteDisponible(quantite)
                .setDisponible(quantite > 0)
                .build());
        responseObserver.onCompleted();
    }

    @Override
    public void decrementerStock(DecrementerRequest request,
                                  StreamObserver<DecrementerResponse> responseObserver) {
        String produitId = request.getProduitId();
        int demande = request.getQuantite();
        Integer actuel = stocks.get(produitId);

        if (actuel == null) {
            responseObserver.onError(Status.NOT_FOUND
                    .withDescription("Produit introuvable : " + produitId)
                    .asRuntimeException());
            return;
        }
        if (actuel < demande) {
            responseObserver.onError(Status.FAILED_PRECONDITION
                    .withDescription("Stock insuffisant : " + actuel + " disponible(s)")
                    .asRuntimeException());
            return;
        }

        int restant = actuel - demande;
        stocks.put(produitId, restant);

        responseObserver.onNext(DecrementerResponse.newBuilder()
                .setSucces(true)
                .setQuantiteRestante(restant)
                .setMessage("Stock mis à jour avec succès")
                .build());
        responseObserver.onCompleted();
    }
}

    • 
      
  • INVALID_ARGUMENTParamètre invalide envoyé par le client (ex: quantité négative). HTTP 400.

06MS Vente — Client gRPC

    
Le microservice vente utilise un stub gRPC — un client généré automatiquement depuis le .proto — pour appeler le service stock sans écrire une ligne de code réseau.


    
Qu'est-ce qu'un stub ?

    
Un stub (ou proxy client généré) est une classe Java produite par le compilateur Protobuf à partir du fichier .proto. Il expose les méthodes du service distant comme de simples appels de méthode Java locaux, en masquant entièrement la complexité réseau sous-jacente.

Côté client (vente), le stub masque entièrement HTTP/2, la sérialisation Protobuf et la gestion du canal réseau :

    

      
Sans stubAvec stub gRPC

      
        

          Ouvrir un socket HTTP/2, sérialiser manuellement en Protobuf binaire, gérer les codes de statut…
          stockStub.verifierStock(request)
        

      
    


    

      💡
      
Stub ≠ mock : Dans le contexte gRPC, un stub est un client généré qui effectue de vrais appels réseau. Ce n'est pas un objet factice de test — pour les tests, on utilise InProcessChannelBuilder (voir section 09).

package edu.ban7.vente.grpc;

import com.example.grpc.stock.StockServiceGrpc;
import io.grpc.ManagedChannel;
import io.grpc.ManagedChannelBuilder;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class GrpcClientConfig {

    @Value("${grpc.client.stock-service.host:localhost}")
    private String host;

    @Value("${grpc.client.stock-service.port:9090}")
    private int port;

    @Bean
    ManagedChannel stockChannel() {
        return ManagedChannelBuilder.forAddress(host, port)
                .usePlaintext()
                .build();
    }

    @Bean
    StockServiceGrpc.StockServiceBlockingStub stockStub(ManagedChannel stockChannel) {
        return StockServiceGrpc.newBlockingStub(stockChannel);
    }
}

    


    
Injection et utilisation du stub

    

      

        JAVA
        vente/src/main/java/edu/ban7/vente/grpc/VenteService.java
        
      

      
package edu.ban7.vente.grpc;

import com.example.grpc.stock.*;
import edu.ban7.vente.exception.*;
import edu.ban7.vente.model.VenteResultat;
import io.grpc.StatusRuntimeException;
import org.springframework.stereotype.Service;

@Service
public class VenteService {

    // Injection par constructeur — le stub est créé par GrpcClientConfig
    private final StockServiceGrpc.StockServiceBlockingStub stockStub;

    public VenteService(StockServiceGrpc.StockServiceBlockingStub stockStub) {
        this.stockStub = stockStub;
    }

    public VenteResultat effectuerVente(String produitId, int quantite) {

        // ── Étape 1 : vérifier le stock ──────────────────────────────────
        StockRequest verifReq = StockRequest.newBuilder()
                .setProduitId(produitId)
                .build();

        StockResponse stockResp;
        try {
            stockResp = stockStub.verifierStock(verifReq);
        } catch (StatusRuntimeException e) {
            throw new ServiceIndisponibleException(
                "Service stock inaccessible : " + e.getStatus().getDescription()
            );
        }

        if (!stockResp.getDisponible()) {
            throw new StockInsuffisantException(
                "Produit " + produitId + " en rupture de stock"
            );
        }

        // ── Étape 2 : décrémenter le stock ───────────────────────────────
        DecrementerRequest decrReq = DecrementerRequest.newBuilder()
                .setProduitId(produitId)
                .setQuantite(quantite)
                .build();

        try {
            DecrementerResponse decrResp = stockStub.decrementerStock(decrReq);

            if (!decrResp.getSucces()) {
                throw new VenteException("Échec décrémentation : " + decrResp.getMessage());
            }

            return VenteResultat.success(produitId, quantite, decrResp.getQuantiteRestante());

        } catch (StatusRuntimeException e) {
            return handleGrpcError(e);
        }
    }

    private VenteResultat handleGrpcError(StatusRuntimeException e) {
        switch (e.getStatus().getCode()) {
            case NOT_FOUND          -> throw new ProduitIntrouvableException(e.getStatus().getDescription());
            case FAILED_PRECONDITION -> throw new StockInsuffisantException(e.getStatus().getDescription());
            case UNAVAILABLE         -> throw new ServiceIndisponibleException("Service stock hors ligne");
            default                  -> throw new VenteException("Erreur gRPC : " + e.getStatus());
        }
    }
}

BlockingStub

        
Synchrone

        
Bloque le thread appelant jusqu'à la réponse. Le plus simple. Recommandé avec Spring WebMVC.

      

      

        
Stub (async)

        
Asynchrone

        
Utilise des callbacks StreamObserver. Non-bloquant, adapté à WebFlux/Reactor.

Retourne un ListenableFuture Guava. Utile pour composer plusieurs appels parallèles.

08Intercepteurs gRPC

    
Les intercepteurs sont l'équivalent des 
filtres Spring MVC pour gRPC. Ils permettent d'ajouter des comportements
 transversaux : logs, authentification, tracing distribué, métriques.


    
Intercepteur serveur (MS Stock)

    

      

        JAVA
        stock/.../interceptor/LoggingServerInterceptor.java
        
      

      
import io.grpc.*;
import net.devh.boot.grpc.server.interceptor.GrpcGlobalServerInterceptor;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

@GrpcGlobalServerInterceptor  // Appliqué automatiquement à tous les services gRPC
public class LoggingServerInterceptor implements ServerInterceptor {

    private static final Logger log = LoggerFactory.getLogger(LoggingServerInterceptor.class);

    @Override
    public <ReqT, RespT> ServerCall.Listener<ReqT> interceptCall(
            ServerCall<ReqT, RespT> call,
            Metadata headers,
            ServerCallHandler<ReqT, RespT> next) {

        String method = call.getMethodDescriptor().getFullMethodName();
        log.info("[gRPC] Appel entrant : {}", method);

        return next.startCall(
            new ForwardingServerCall.SimpleForwardingServerCall<>(call) {
                @Override
                public void close(Status status, Metadata trailers) {
                    log.info("[gRPC] Réponse {} : status={}", method, status.getCode());
                    super.close(status, trailers);
                }
            }, headers
        );
    }
}

    


    
Intercepteur client (MS Vente)

    

      

        JAVA
        vente/.../interceptor/AuthClientInterceptor.java
        
      

      
import io.grpc.*;
import net.devh.boot.grpc.client.interceptor.GrpcGlobalClientInterceptor;

@GrpcGlobalClientInterceptor  // Appliqué à tous les canaux gRPC du client
public class AuthClientInterceptor implements ClientInterceptor {

    @Override
    public <ReqT, RespT> ClientCall<ReqT, RespT> interceptCall(
            MethodDescriptor<ReqT, RespT> method,
            CallOptions callOptions,
            Channel next) {

        return new ForwardingClientCall.SimpleForwardingClientCall<>(
                next.newCall(method, callOptions)) {

            @Override
            public void start(Listener<RespT> responseListener, Metadata headers) {
                // Ajouter un header d'authentification à chaque appel gRPC
                Metadata.Key<String> authKey = Metadata.Key.of(
                    "x-service-token", Metadata.ASCII_STRING_MARSHALLER
                );
                headers.put(authKey, "mon-token-interne-secret");
                super.start(responseListener, headers);
            }
        };
    }
}

    


    

      ✅
      
Ordre des intercepteurs : Les intercepteurs s'appliquent dans l'ordre de leur déclaration. Pour contrôler l'ordre, implémentez Ordered ou utilisez @Order(n) sur la classe.

    

  


  
  

    
09Tests & outils

    
gRPC propose des utilitaires dédiés pour les tests unitaires (serveur en mémoire) et grpcurl pour les tests manuels en ligne de commande.


    
Test unitaire avec serveur en mémoire

    

      

        JAVA
        stock/.../grpc/StockGrpcServiceTest.java
        
      

      
import io.grpc.inprocess.*;
import io.grpc.testing.GrpcCleanupRule;
import org.junit.*;
import static org.assertj.core.api.Assertions.*;

class StockGrpcServiceTest {

    @Rule
    public final GrpcCleanupRule grpcCleanup = new GrpcCleanupRule();

    private StockServiceGrpc.StockServiceBlockingStub stub;

    @BeforeEach
    void setUp() throws Exception {
        String serverName = InProcessServerBuilder.generateName();

        // Serveur en mémoire — pas de réseau réel
        grpcCleanup.register(
            InProcessServerBuilder.forName(serverName)
                .directExecutor()
                .addService(new StockGrpcService(new StockServiceMock()))
                .build().start()
        );

        stub = StockServiceGrpc.newBlockingStub(
            grpcCleanup.register(
                InProcessChannelBuilder.forName(serverName)
                    .directExecutor().build()
            )
        );
    }

    @Test
    void verifierStock_retourneLaQuantiteDisponible() {
        StockRequest req = StockRequest.newBuilder()
                .setProduitId("PROD-001").build();

        StockResponse resp = stub.verifierStock(req);

        assertThat(resp.getQuantiteDisponible()).isEqualTo(10);
        assertThat(resp.getDisponible()).isTrue();
    }

    @Test
    void verifierStock_produitInconnu_leveNotFound() {
        StockRequest req = StockRequest.newBuilder()
                .setProduitId("INCONNU").build();

        assertThatThrownBy(() -> stub.verifierStock(req))
            .isInstanceOf(StatusRuntimeException.class)
            .extracting(e -> ((StatusRuntimeException) e).getStatus().getCode())
            .isEqualTo(Status.Code.NOT_FOUND);
    }
}
